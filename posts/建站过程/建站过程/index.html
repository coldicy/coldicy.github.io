建站过程

2025-02-27

先根据paper-mode官方文档搭建基础博客

https://github.com/adityatelange/hugo-PaperMod/wiki/Installation

创建博客站点

hugo new site hugo-blog --format yaml

初始化本地git仓库

cd hugo-blog

git init
git add .
git commit -m "init"

使用Git Submodule方式安装PaperMod主题

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

后续配置参考

https://sonnycalcr.github.io/posts/build-a-blog-using-hugo-papermod-github-pages/

修改hugo.yaml文件

baseURL: https://coldicy.github.io/ # 主站URL,改成自己的
languageCode: zh-cn # 语言
title: Build Test # 网站标题
theme: PaperMod # 主题
copyright: "[© Licensed under CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/legalcode.zh-hans)" # 网站的版权声明，显示在页脚
pagination:
    pagerSize: 5 # 博客每一页显示文章的数量

本地启动博客

hugo server

启动后访问控制台显示的地址即可看见刚刚搭建的博客（当然现在什么也没有）

创建第一篇文章（这个命令很重要，每次新建文章只需修改 test.md 为你文章的名字即可）

hugo new posts/test.md

默认情况下通过 hugo server 构建的博客站点并不会将刚刚创建的文章显示在页面上，需将文章前言部分 draft: true 改为 draft: false 才行，这也意味着你可以通过草稿功能控制你想在页面上展示哪些文章

如果不想区分草稿，也可在构建博客时使用 hugo server -D 命令，-D 意味着在构建时包括草稿。此时重新构建博客即可看到我们的文章了
